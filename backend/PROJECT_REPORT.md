## 1. Project Overview
ContextPilot is an AI-powered browser assistant backend designed to provide contextual help, summaries, and specialized tasks based on the user's current webpage content. It features a multi-provider AI architecture (Anthropic/Gemini) and uses Supabase for persistent session management, user profiling, and feedback loops.

## 2. Technical Stack
- **Runtime:** Node.js (>=20)
- **Framework:** Express.js (v5.2.1)
- **AI Providers:** 
    - **Anthropic:** Claude 3.5 Sonnet (Default)
    - **Google Gemini:** Gemini 2.5 Flash (Verified and working)
- **Database:** Supabase (PostgreSQL with Realtime capabilities)
- **Environment Management:** dotenv
- **Testing:** Bash-based curl test suites (`test-api.sh`)

## 3. Core Architecture & Modules

### A. AI Orchestration (`agent/`)
- **`orchestrator.js`**: Manages the agent loop, including prompt building, calling the AI provider, and handling tool use (multi-turn logic).
- **`promptBuilder.js`**: Dynamically constructs system prompts based on user preferences and historical feedback/corrections stored in Supabase.
- **`tools.js`**: Defines the agent's available toolset (JSON schemas for tool definitions).
- **`toolExecutor.js`**: Currently a stub-based execution layer for specialized tasks like summarization, email drafting, etc.

### B. Library Layer (`lib/`)
- **`anthropic.js`**: Unified wrapper for both Anthropic and Gemini APIs. Includes robust fallback logic for Gemini models.
- **`supabase.js`**: Service layer for database operations (saving messages, fetching preferences, recording feedback).
- **`config.js`**: Centralized configuration management and environment variable validation.

### C. API Routes (`routes/`)
- **`/api/chat`**: Main entry point for AI interactions. Truncates content based on `MAX_PAGE_CONTENT_CHARS`.
- **`/api/feedback`**: Records user ratings and corrections for agent learning.
- **`/api/history`**: Retrieves session-based chat history.
- **`/health`**: Basic service health check.
